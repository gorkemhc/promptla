N8N OTOMASYON REHBERİ

Uygulamaları, API’leri ve AI modellerini görsel workflow’larla bağla.

Node, trigger, credential, webhook, HTTP Request, AI Agent, insan onayı, hata yönetimi ve güvenli yayınlama mantığı adım adım açıklanır.

Seviye: Başlangıç → İleriYerel öğrenme: localhostSon kontrol: 21 Temmuz 202610 hazır akış örneği

n8n nedir?

n8n, bir olay gerçekleştiğinde veriyi alıp bir dizi node üzerinden işleyen görsel otomasyon platformudur. Form, webhook, zamanlayıcı veya uygulama olayı workflow’u başlatabilir; sonraki node’lar veriyi dönüştürür, API çağrısı yapar, AI modeline gönderir veya başka bir sisteme yazar.

i
Workflow = adımların bütünü

Bir node tek işi yapar. Güvenilir otomasyon, her node’un girişini ve çıktısını açık tutar.

Yerel öğrenme ortamı

n8n’i öğrenmek için yerel bilgisayarda çalıştırabilirsin. Bu yöntem dışarıdan erişilebilir webhook gerektiren gerçek yayın senaryoları için tek başına yeterli değildir; ancak node ve veri akışını öğrenmek için uygundur.

Kurulum yapmadan deneme
node --version
npm --version
npx n8n
Global npm kurulumu
npm install -g n8n
n8n start
i
Yerel adres

Başladıktan sonra editör çoğunlukla http://localhost:5678 adresinde açılır. Terminali kapatırsan süreç durur.

İlk workflow: Webhook → yanıt

Yeni workflow oluştur

Boş canvas aç ve anlaşılır bir isim ver.

Webhook node ekle

Test URL ile POST isteği kabul et.

Set/Edit Fields node ekle

Gelen veriden yalnızca gerekli alanları seç.

Respond to Webhook ekle

Kullanıcıya kontrollü JSON yanıtı döndür.

Test et

Örnek veri gönder, her node’un çıktısını ayrı ayrı incele.

Production URL’ye geç

Workflow’u aktive ettikten sonra üretim URL’sini kullan.

Örnek POST gövdesi
{
  "name": "Görkem",
  "task": "Promptumu iyileştir"
}

HTTP Request ile API bağlantısı

HTTP Request node, REST API’lerine GET, POST, PUT, PATCH veya DELETE isteği göndermek için kullanılır. URL, method, header, query ve body alanlarını API dokümanına göre yapılandır.

Kimlik doğrulamayı seç

Mümkünse credential alanını kullan; anahtarı node içine düz metin yazma.

Method ve URL

API sözleşmesindeki endpoint ile eşleştir.

Girdi verisini bağla

Önceki node alanlarını expression ile body’ye yerleştir.

Hata durumunu tasarla

4xx ve 5xx sonuçlarında yeniden deneme veya hata akışı belirle.

AI Agent nasıl çalışır?

AI Agent node, dil modelini bir veya daha fazla araçla birleştirerek hangi aracı ne zaman kullanacağına karar verebilir. Basit metin üretimi için agent kullanmak şart değildir; yalnızca araç seçimi veya çok adımlı karar gerektiğinde kullan.

Claude, OpenAI, Codex ve GitHub bağlantıları

n8n içinde model sağlayıcı node’ları veya HTTP Request ile API çağrıları kullanılabilir. Codex ve Claude Code ise çoğunlukla geliştirici iş akışında repository üzerinde çalışır; n8n, issue oluşturma, görev kuyruğu, onay, bildirim ve raporlama katmanını yönetebilir.

Hata ve yeniden deneme tasarımı

Gerçek otomasyonlarda ağ kesintisi, kota, yetki ve veri biçimi hataları normaldir. Başarılı akış kadar hata akışı da tasarlanmalıdır.

Timeout ve retry sınırı belirle
Aynı işlemin iki kez çalışmasını önle
Hatalı kaydı ayrı yere yaz
Kritik eylemde insan onayı kullan
Kişisel veriyi loglarda maskele
Test ve production credential’larını ayır

Güvenlik kontrol listesi

  • n8n editörünü doğrudan internete açık ve şifresiz bırakma.
  • Credential’ları node içine düz metin olarak koyma.
  • Webhook girişlerini doğrula; beklenmeyen alanları reddet.
  • AI çıktısını SQL, e-posta, ödeme veya silme işlemine doğrudan bağlama.
  • Workflow JSON’unu paylaşmadan önce credential ve kişisel veri kontrolü yap.
  • Yerleşik güvenlik denetimini ve güncel sürüm notlarını takip et.
