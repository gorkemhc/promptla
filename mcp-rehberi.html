MCP nedir?

Model Context Protocol, AI uygulamaları ile harici veri ve araçlar arasında ortak bir bağlantı yapısı tanımlar. Amaç, her entegrasyon için tamamen farklı yöntem yazmak yerine istemci ve sunucuların aynı protokol üzerinden anlaşmasıdır.

i
MCP bir model değildir

MCP, modelin hangi araçları görebileceğini ve nasıl çağıracağını taşıyan bağlantı katmanıdır.

Yerel ve uzak bağlantı

Yerel MCP sunucuları çoğunlukla aynı bilgisayarda süreç olarak çalışır ve STDIO üzerinden iletişim kurabilir. Uzak sunucular ağ üzerinden Streamable HTTP gibi bir taşıma kullanabilir.

Yerel: düşük gecikme, cihaz izinleri önemli
Uzak: kimlik doğrulama ve TLS önemli
Her iki durumda da araç listesi kontrol edilmeli
Gereksiz veri alanları modele gönderilmemeli

Claude Code ile MCP

Claude Code’a bir MCP sunucusu eklediğinde, sunucunun sunduğu araçlar oturum içinde kullanılabilir hâle gelir. Kurulum komutları sunucunun resmi dokümantasyonuna göre değişir.

Sunucuyu doğrula

Resmi repository, yayıncı ve paket adını kontrol et.

Yetkileri oku

Dosya, ağ, GitHub veya veri tabanı erişimini incele.

Kısıtlı kapsamla ekle

Önce salt-okunur veya test hesabı kullan.

Araç listesini kontrol et

Beklemediğin tool varsa bağlantıyı kapat.

Küçük test yap

Gerçek veriyi değiştirmeyen bir sorguyla başla.

!
Komutları kaynaktan al

MCP sunucusu kurulum komutları hızlı değişebilir. Bu nedenle komutu sunucunun güncel ve resmi dokümanından al.

n8n MCP bağlantısı

n8n, etkinleştirilen workflow’ları MCP üzerinden araç olarak sunabilen bir MCP server özelliğine sahiptir. Böylece MCP istemcisi, izin verilen workflow’ları çağırabilir.

n8n MCP erişimini etkinleştir

Instance ayarlarından MCP bağlantısını aç.

Workflow’u uygun hâle getir

Açık giriş şeması, kontrollü çıktı ve güvenli credential kullan.

Yalnızca gerekli workflow’u yayınla

Bütün otomasyonları araç olarak açma.

MCP istemcisine bağlantı ekle

n8n’in verdiği endpoint ve kimlik bilgilerini kullan.

Çağrıyı logla ve sınırla

Kimin, ne zaman, hangi parametrelerle çalıştırdığını izle.

×
Kritik workflow’larda onay

Para gönderme, veri silme veya dışarıya e-posta gönderme gibi eylemleri doğrudan model kararına bırakma.

Örnek entegrasyon senaryosu

Claude Code → n8n → GitHub

Kullanıcı Claude Code’dan “bu hatayı ekip görevine dönüştür” ister. Claude Code, MCP aracı üzerinden n8n workflow’unu çağırır. n8n girdiyi doğrular, GitHub issue oluşturur ve Slack bildirimi gönderir.

Claude Code→MCP Tool→n8n Workflow→GitHub Issue→Slack
i
Girdi şeması kullan

Tool parametrelerinde başlık, açıklama ve öncelik gibi alanları açıkça tanımlamak hatalı çağrıları azaltır.

MCP güvenlik kontrol listesi

  • Sunucuyu ve paket kaynağını doğrula.
  • Sunucunun sunduğu tüm tool adlarını ve açıklamalarını oku.
  • Dosya sistemi erişimini proje klasörüyle sınırla.
  • Production credential yerine test hesabıyla başla.
  • Tool çağrılarını ve hataları logla.
  • İnsan onayı gerektiren eylemleri ayrı workflow’a taşı.
  • Kullanılmayan MCP bağlantılarını kapat.
  • Prompt injection ile aracın yanlış kullanılabileceğini varsay.
